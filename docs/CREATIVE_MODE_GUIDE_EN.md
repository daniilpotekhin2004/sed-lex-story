# Guide to creative mode

This document describes exactly the “creative mode” of the project: a fast authoring flow in which there is no need to build a large library of assets in advance, make a full package of character references, or generate locations before assembling the plot.

Key idea of ​​the mode:

- first collect a playable version of the story;
- then, if necessary, increase the quality of assets;
- do not slow down work due to preliminary preparation of the library.

In this mode you can work like this:

- create a project;
- create only the necessary characters;
- create simple text locations or add them as needed;
- immediately go to the graph and write scenes;
- quickly check the result in the player.

![Project Screen](images/quest-guide-ru/02-project-detail.png)

---

## What is creative mode

`Creative Development` is a simplified mode of the project, in which the focus is shifted from preparing the full production package to creating the story itself.

What does this mean in practice:

- you don't need a mandatory external 'studio library' to get started;
- you do not need to generate location sheets in advance;
- you don’t need a full set of character refs before the first working version;
- locations can be kept as scene context and text cards;
- the main emphasis is on the plot, scenes, elections and quick viewing of the result.

What is especially important:

- locations in creative mode do not require a full set of references;
- for characters, a basic minimum is enough, and not a full review package;
- the project library in this mode is reduced to key entities: `Characters` and `Locations`.

---

## How creative mode differs from standard mode

In standard mode, a more production approach is usually expected:

- more tabs and entities;
- more rigorous preparation of assets;
- pre-collected locations, documents, artifacts;
- a complete package of visual supports for generation.

In creative mode the logic is different:

- first the story, then the packaging;
- first the scene and meaning, then the complete visual set;
- the project library is not required as a starting point;
- location generation does not need to be done in advance;
- characters do not need full multiview until the first playable version.

In short:

- `standard` - first production preparation;
- `creative` - first the author's draft, playable and testing the idea.

---

## When creative mode works best

Use it if:

- you need to quickly assemble a new story from scratch;
- you are still looking for the structure of scenes and branches;
- I don’t want to waste time on fully preparing assets in advance;
- you want to first check whether the dramaturgy itself works;
- you only have some visuals or don’t have them at all yet.

This is the best mode for:

- first prototypes;
- pilot training quests;
- short interactive stories;
- rough assembly before more serious production.

---

## What you don’t have to do in advance in this mode

This is the main list of what can be safely postponed until the next stage.

### It is not necessary to use the studio library

You don't need to first go to the general studio and collect a database of characters and locations there.

You can work directly inside the project:

- open `Creative Development`;
- create a character directly in the project;
- create a location directly in the project;
- go to the graph and continue writing the quest.

Importing from the library remains available, but it is optional, not required.

### It is not necessary to generate locations in advance

In creative mode, a location is not required to have the full package:

- `Exterior`;
- `Interior`;
- `Detail';
- `Map`.

You can:

- leave only the name and description of the location;
- if desired, upload one main image;
- do not make a location sheet at all until it is needed.

That is, a location in this mode can exist as a scene context, and not as a pre-packaged asset.

### It is not necessary to do a full character review in advance.

In creative mode, the character does not need a full package of angles and turns at the start.

For the first working version, the minimum is enough:

- `Portrait`;
- `Full growth`.

This is enough to:

- fix the hero;
- tie it to scenes;
- move further along the plot.

All additional angles, poses and extended references can be added later.

## Fast route in creative mode

If you want the shortest working circuit, use this order:

1. Create a project.
2. Open `Creative Development`.
3. Create only key characters.
4. Create only the necessary locations in the form of short cards.
5. Go to graph.
6. Collect scenes and branches.
7. Complete the quest in the player.
8. Only then decide which assets need to be strengthened.

This gives maximum speed and minimum entry threshold.

---

## Step 1: Create or open a project

1. Go to `Projects`.
2. Create a new project or open an existing one.
3. In the project card, click `Creative Development`.

If you find yourself in the 'World Library', check the mode switch and make sure 'Creative Development' is selected.

A sign that you are in the right mode:

- the interface is simplified;
- main focus on the `Characters` and `Locations` tabs;
- there is no requirement to assemble a complete production package in advance.

---

## Step 2: Create only the main characters first

In creative mode, you don't need to start all the characters in the world at once.

Correct approach:

- start with 1-3 really important heroes;
- add others only when they are needed in scenes.

Practice:

1. Open the `Characters` tab.
2. Press `+ Quick` if the character is already clear.
3. Use `+ Master` if the image still needs to be expanded in more detail.
4. Fill in the minimum:
- Name;
- brief description;
- role;
- appearance;
- voice or manner of speech.

![Quick character creation](images/quest-guide-ru/04-character-quick-create.png)

What is important specifically for creative mode:

- do not try to immediately make an ideal character encyclopedia;
- such a description is enough to understand who it is and why he is in the scene;
- if a character does not fall into the next scenes, do not waste time on fully developing him.

---

## Step 3. Basic refs are enough for characters

This is one of the most important principles of creative mode.

The rough working version does not require a full package of angles. Usually it is enough:

- `Portrait`;
- `Full growth`.

That is why it is convenient in this mode:

- upload your two images;
- either quickly finish off what is missing in batches;
- do not stop for side and rear angles.

![Character in creative mode](images/quest-guide-ru/05-character-assets.png)

Practical recommendations:

- if you have your own images, just upload them to the required slots;
- if you don’t have your own images, don’t try to make a full set right away;
- if there are several characters, first give basic refs to those participating in the nearest scenes.

Good working minimum:

- 2 characters with basic refs;
- clear roles;
- enough material to tie them to the scenes.

---

## Step 4. Locations can be kept simple and text-based

In many cases, it is the locations that prevent a quick start: the author wants to collect a beautiful set of cards, diagrams and views in advance. This is not necessary in creative mode.

Correct approach:

- create a location as a simple entity;
- record the name and description;
- add one image if desired;
- do not make a sheet in advance if it is not needed for the current scene.

What you can fill out right away:

- `Name`;
- `Description`;
- `Visual description`;
- ``Last name''.

![Quick location creation](images/quest-guide-ru/06-location-quick-create.png)

After this, you can already use the location in scenes.

If you want, you can additionally:

- upload one preview;
- add references later;
- make a location sheet later.

But this should not block work on the story.

![Location in creative mode](images/quest-guide-ru/07-location-assets.png)

Main principle:

- in creative mode, the location does not have to be production-ready so that you can write and test the scene.

---

## Step 5. Go straight to the graph

Once you have:

- main plot;
- key characters;
- at least simple locations;

you can go to `Graph`.

This is where creative mode shines best because the focus shifts to:

- sequence of scenes;
- player decisions;
- branching logic;
- quick view of the result.

![Graph in creative mode](images/quest-guide-ru/09-graph-editor.png)

What to do in the column:

1. Create a project graph.
2. Create scenes.
3. Fill in for each scene:
- title;
- synopsis;
- content;
- type (`History` or `Decision`).
4. Tie down the characters if they are participating in the scene.
5. If necessary, select a location.
6. Create transitions between scenes.
7. Run validation.

In creative mode, it's important to remember two things here:

- the scene is more important than a complete set of assets;
- the logic of choice is more important than complete visual coverage.

---

## How to properly work with scenes in creative mode

The most working approach is this:

###Text and choices first

First, make it clear:

- what happens in the scene;
- why the player makes a decision here;
- what each option leads to.

### Then the characters

When the scene is already clear in meaning:

- link the necessary characters;
- if necessary, clarify the context in the scene for them.

### Then the location

If the scene is location specific, choose a location. But if the location has not yet been fully determined, it is not necessary to immediately turn it into a full-fledged visual package.

### And only then - visual enhancement

When the scene is already running, you can decide:

- is a separate location preview necessary?
- is an additional character reference needed?
- whether it is necessary to enhance a specific frame.

This saves a lot of time.

---

## What should not slow down the assembly of the first version

In Creative mode, you shouldn't let these tasks block your progress:

- “first I’ll collect the entire project library”;
- “first I’ll make a full package of refs for each hero”;
- “first I’ll prepare all the locations with a map and details”;
- “First I’ll finish drawing everything, then I’ll start writing scenes.”

All these tasks can be postponed.

For the first playable version you really only need:

- working structure of scenes;
- clear transitions;
- key characters;
- at least minimal visual supports.

---

## Step 6. Check the history in the player as early as possible

Once the scenes and transitions are assembled, don't put off testing.

Need to:

1. Return to the project.
2. Publish the release.
3. Open `Player`.
4. Complete the story as a regular player.

![Player: quick draft review](images/quest-guide-ru/11-player-preview.png)

In Creative mode, the player is especially important because it quickly shows:

- where the scene falls apart logically;
- where the choice is poorly formulated;
- where there is a lack of explanation;
- where the character appears too early or too late;
- where the location is not yet needed as a full-fledged asset, and where it is already needed.

It is after this test that it becomes clear what is really worth strengthening.

---

## Ideal workflow in creative mode

Here is the order that gives the best results:

1. The idea of ​​history.
2. 3-6 scenes of the draft.
3. 1-3 key characters.
4. 1-3 simple locations.
5. Graph assembly.
6. Publication.
7. Run in the player.
8. Logic correction.
9. Only then - strengthening of visuals, locations and refs.

If you do the opposite, you almost always waste more time and get less benefit.

---

## When is it worth returning to the library and the full package of assets

Once the draft is already working, you can move on to the next level.

It is worth returning to a more complete production approach if:

- the story has already passed playtest;
- the structure of scenes has stabilized;
- branches no longer change every hour;
- it became clear which characters and locations are truly key.

Only at this stage does it make sense:

- expand the project library;
- collect location references;
- make a full set of character angles;
- move to a more rigid production pipeline.

---

## The most common mistake in creative mode

The error looks like this:

- the author turns on creative mode,
- but continues to work as if it were standard mode,
- that is, he still tries to make a complete set of everything in advance.

As a result, creative mode loses its meaning.

The correct thinking here is different:

- I don’t prepare the whole world in advance;
- I'm putting together a working story;
- I only strengthen what is really needed after checking.

---

## Short checklist for creative mode

Before the first check in the player, it is enough to do this:

- the project has been created;
- “Creative development” mode is open;
- there are at least 1-3 characters;
- important characters have a basic minimum of refs;
- there are 1-3 locations in a simple form or at least the context of the scenes is selected;
- the graph has been assembled;
- transitions have been made;
- release published;
- the story follows at least one main branch.

If all this is present, then creative mode worked correctly.

---

## Brief conclusion

Creative mode is needed not for the ideal preparation of assets, but for a quick authoring cycle:

- came up with;
- collected;
- checked;
- corrected;
- only then I enhanced the visual part.

The main rule of the regime:

- it is not the library that controls history,
- and history tells us what assets are really needed.

This is why it is not required here:

- use the library as a mandatory start;
- generate all locations in advance;
- collect a complete overview of the characters in advance.
